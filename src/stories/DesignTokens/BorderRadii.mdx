# Border Radii

NSX uses a consistent border radius scale to create cohesive rounded corners across all interface elements.

## Border Radius Scale

From sharp corners to fully rounded, designed for different UI elements and visual styles.

## Usage Guidelines

### Border Radius Selection

- **rounded-sm (2px)**: Small buttons, compact UI elements, technical interfaces
- **rounded-md (6px)**: Default for most components - forms, buttons, small cards
- **rounded-lg (8px)**: Cards, containers, dropdowns, most prominent elements
- **rounded-xl (12px)**: Large containers, featured content, modals
- **rounded-2xl+ (16px+)**: Marketing elements, hero sections, highly prominent features
- **rounded-full**: Circular elements, avatars, badges, pill-shaped buttons

### Consistency Rules

- **Forms**: Use `rounded-md` (6px) for consistency across inputs, selects, textareas
- **Buttons**: Use `rounded-sm` for compact buttons, `rounded-md` for standard buttons
- **Cards**: Use `rounded-lg` (8px) as the standard, `rounded-xl` for featured content
- **Navigation**: Match border radius to the overall design language of your nav style

### Accessibility

- Border radius doesn't affect accessibility directly
- Ensure sufficient contrast is maintained around rounded corners
- Consider that very large border radius values can affect button/link target size

### Performance

- Border radius is efficiently rendered by modern browsers
- Avoid animating border-radius changes for better performance
- Use consistent values to enable better CSS optimization

### Common Patterns

- **Button Groups**: Use directional radius (`rounded-l-md rounded-r-none`) for connected buttons
- **Card Headers/Footers**: Use `rounded-t-lg` and `rounded-b-lg` for sectioned cards
- **Tabs**: Use `rounded-t-md` for tab interfaces
- **Dropdowns**: Match the border radius of the trigger element for visual connection
