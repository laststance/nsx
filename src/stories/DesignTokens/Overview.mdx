# Design Tokens Overview

NSX uses a comprehensive design token system built on Tailwind CSS v4 to ensure consistency, maintainability, and scalability across the entire application.

## What are Design Tokens?

Design tokens are the atomic design decisions that define your design system. They are named entities that store visual design attributes like colors, spacing, typography, and more. In NSX, these tokens serve as the single source of truth for all design decisions.

## Implementation

### Tailwind CSS v4 Configuration

NSX uses Tailwind CSS v4's new `@theme` directive to define all design tokens in the `global.css` file:

```css
@theme {
  /* Colors */
  --color-primary-500: #22c55e;
  --color-secondary-500: #3b82f6;

  /* Spacing */
  --spacing-4: 1rem;
  --spacing-6: 1.5rem;

  /* Typography */
  --font-size-base: 1rem;
  --font-weight-semibold: 600;

  /* And more... */
}
```

## Benefits

### Consistency

- Unified visual language across all components and pages
- Consistent spacing, colors, and typography throughout the application
- Reduced design debt and visual inconsistencies

### Maintainability

- Single source of truth for all design decisions
- Easy to update colors, spacing, or typography across the entire application
- Simplified theme switching and customization

### Developer Experience

- Predictable class names and utilities
- IntelliSense support in modern editors
- Clear documentation and examples for all tokens

### Scalability

- Easy to extend with new tokens or modify existing ones
- Systematic approach to design that grows with the application
- Clear patterns for adding new components

### Accessibility

- Built-in contrast ratios and accessibility considerations
- Consistent focus states and interactive element sizing
- Responsive design patterns that work across all devices

## Best Practices

1. **Use Semantic Utilities**: Prefer `bg-primary` over `bg-white` for better theme support
2. **Stick to the Scale**: Use defined spacing and sizing values rather than arbitrary ones
3. **Layer Consistently**: Use shadows to create clear visual hierarchy
4. **Test Responsively**: Ensure your usage works across all breakpoints
5. **Follow Patterns**: Use established component patterns for consistency
