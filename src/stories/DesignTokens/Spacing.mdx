# Spacing

NSX uses a consistent 4px-based spacing scale for padding, margins, gaps, and positioning.

## Spacing Scale

## Usage Guidelines

### Spacing Scale

- **0-1**: Use for very tight spacing, icon padding, borders
- **2-3**: Use for compact layouts, form element spacing
- **4**: Most common spacing value, standard element spacing
- **6**: Use for section spacing, card padding
- **8**: Use for large element spacing, page sections
- **10+**: Use for major layout spacing, page-level separation

### Common Patterns

- **Buttons**: `py-2 px-4` (8px vertical, 16px horizontal)
- **Form Inputs**: `px-3 py-2` (12px horizontal, 8px vertical)
- **Cards**: `p-4` or `p-6` (16px or 24px all around)
- **Page Sections**: `py-8` or `py-12` (32px or 48px vertical)
- **Navigation**: `px-4 py-2` (16px horizontal, 8px vertical)

### Responsive Spacing

- Use responsive prefixes (`sm:`, `md:`, `lg:`) to adjust spacing at different breakpoints
- Smaller screens typically need less spacing
- Larger screens can accommodate more generous spacing

### Consistency

- Stick to the defined scale - avoid arbitrary values
- Use consistent spacing patterns across similar components
- Align spacing with the 4px grid system
- Use space between utilities (`space-x-*`, `space-y-*`) for consistent element spacing
