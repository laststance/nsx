## Usage Guidelines

### Shadow Hierarchy

- **shadow-xs/sm**: Use for subtle elevation, buttons, form elements
- **shadow-md**: Use for cards, panels, content containers
- **shadow-lg**: Use for dropdowns, tooltips, popovers that appear above content
- **shadow-xl/2xl**: Use for modals, overlays, prominent floating elements
- **shadow-inner**: Use for input fields, pressed states, inset elements

### Best Practices

- **Layer Consistently**: Use shadows to create clear visual hierarchy
- **Less is More**: Don't overshadow (pun intended) - use shadows purposefully
- **Context Matters**: Floating elements need stronger shadows than embedded elements
- **Accessibility**: Ensure shadows don't interfere with focus indicators
- **Performance**: Use box-shadow efficiently - avoid animating large shadows

### Common Patterns

- **Form Focus**: Combine shadow-xs with colored border for focus states
- **Button Hover**: Slightly increase shadow on hover for interactive feedback
- **Card Hover**: Enhance shadow-md to shadow-lg on hover for interactivity
- **Layered Menus**: Use progressively stronger shadows for nested menu levels

### Dark Mode Considerations

- Shadows are less visible on dark backgrounds
- Consider using border or background color changes as alternatives
- If using shadows on dark backgrounds, increase opacity or use colored shadows
